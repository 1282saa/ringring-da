## 분석 개요

### 목적
사용자들이 어떤 서비스에서 어떤 서비스로 이동하는지 패턴 분석
→ "스픽에서 링글로 온 이유는?", "링글에서 어디로 가는가?"

### 방법
1. 각 서비스 리뷰에서 경쟁사 브랜드명 언급 탐지
2. 전환 패턴 키워드 탐지: "갈아탔다", "옮겼다", "바꿨다" 등
3. 서비스 간 언급 관계 매트릭스 생성

### 전환 패턴 분석

#### 링글로 오는 사용자 (유입 패턴)
```
캠블리 → 링글: "캠블리 하다가 링글로 갈아탔어요"
         → 이유: 튜터 품질, 명문대 튜터, 비즈니스 영어

스픽 → 링글: "스픽으로 기초 다지고 링글로 넘어왔어요"
       → 이유: 실제 대화 경험, 피드백 품질
```

#### 링글에서 나가는 사용자 (이탈 패턴)
```
링글 → 스픽: "가격 부담으로 스픽으로 갈아탔어요"
       → 이유: 가격, AI 무제한 연습

링글 → 캠블리: "좀 더 캐주얼한 대화 원해서"
        → 이유: 예약 유연성, 가격
```

## 링글 AI를 위한 시사점

### 1. 스픽 대응 전략
**스픽의 강점**: AI 무제한, 저가, 24시간
**링글 AI 대응**:
- AI 튜터로 "무제한 발화" 기능 제공
- "AI로 연습 → 튜터로 피드백" 하이브리드 모델
- "스픽보다 똑똑한 AI" 품질 차별화

### 2. 캠블리 대응 전략
**캠블리의 강점**: 즉시 연결, 유연성
**링글 AI 대응**:
- AI 튜터 = 예약 불필요, 즉시 시작
- 24시간 가능 강조

### 3. 이탈 방지 전략
| 이탈 이유 | AI 솔루션 |
|-----------|-----------|
| 가격 부담 | AI 단독 저가 상품 |
| 예약 어려움 | AI는 예약 불필요 |
| 발화량 부족 | AI로 무제한 연습 |
| 앱 오류 | 앱 품질 개선 (별도) |

### 4. 유입 강화 전략
| 유입 이유 | 강화 방안 |
|-----------|-----------|
| 튜터 품질 | AI + 명문대 튜터 시너지 |
| 비즈니스 영어 | AI 비즈니스 시나리오 강화 |
| 회사 지원 | B2B AI 상품 개발 |
